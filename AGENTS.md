# FW Blog — Agent Guide

Personal blog at [imfw.io](https://imfw.io), a static site built with Eleventy 3 and Tailwind CSS v4, deployed to GitHub Pages.

## Hard constraints

- MUST NOT create `tailwind.config.js` or `postcss.config.js` — Tailwind v4 is configured in CSS via `@import "tailwindcss"` / `@plugin` in `src/assets/css/input.css` (source: package.json `@tailwindcss/cli` v4)
- MUST NOT edit `_site/` or `.cache/` by hand — build output, regenerated by `npm run build` (source: eleventy.config.mjs `dir.output`)
- MUST NOT remove or bypass the custom `slug` filter — it transliterates Chinese tags to pinyin; the built-in slugifier strips CJK and produces empty/duplicate tag URLs (source: eleventy.config.mjs `toSlug`)
- New posts MUST go under `src/posts/<year>/`; do NOT set `layout` or a `posts` tag in post frontmatter — `src/posts/posts.json` directory data supplies both (source: src/posts/posts.json)
- MUST NOT break the `CNAME` passthrough copy — the custom domain dies without it (source: eleventy.config.mjs, deploy.yml `cname: imfw.io`)
- Pushing to `main` deploys straight to production — MUST see `npm test` pass locally first (source: .github/workflows/deploy.yml)

## Commands

```bash
npm install        # install dependencies
npm start          # dev server at http://localhost:8080 + Tailwind watch
npm test           # vitest: filter units + content checks + build smoke — the verification gate for "done"
npm run build      # production build (11ty then Tailwind) into _site/
npm run clean      # remove _site/
```

There is no linter yet.

## Session routine

- **Clock-in:** read [PROGRESS.md](PROGRESS.md) → `git log -3` + `git status` → run
  `npm test` → pick up the single active item (WIP = 1).
- **Clock-out:** `npm test` passes → update PROGRESS.md (state, commit hash,
  test status) → remove stale artifacts (debug logs, commented-out code) → commit.
  Session complete = task verified AND clean state — not before.

## Conventions

- Site-visible text (nav, page titles, date format) is Traditional Chinese; code comments in `eleventy.config.mjs` and `input.css` are also zh-TW — keep both when editing.
- Global site metadata lives in `src/_data/site.json`; templates read it as `{{ site.* }}` — never hardcode the title, URL, or author in templates.
- Post filenames follow `YYYY-MM-DD-slug.md`; the date prefix must match the frontmatter `date` (enforced by tests/content.test.mjs).
- Tags that slugify to the same URL (`Docker`/`docker`, homophonous Chinese words) or to an empty string fail the build — rename the tag (source: lib/filters.mjs `assertNoSlugCollisions`).

## Docs maintenance

When modifying any file under `docs/`, update its `> **Last updated:** YYYY-MM-DD`
frontmatter to today's date. Requirement keywords (MUST, SHOULD, MAY) follow RFC 2119.
